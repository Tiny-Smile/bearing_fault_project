# Git Commit Output

## 触发时机

当用户请求并完成以下任一任务后，必须应用本技能：

- 改代码
- 写代码
- 调代码
- 修复 bug
- 重构或优化

仅在“任务完成、准备给出最终答复”时输出提交信息。

## 输出规则

1. 最后一行固定输出 Git 提交信息。
2. 类型必须从以下集合中自动选择：`feat` / `fix` / `refactor` / `style` / `chore`。
3. 描述必须简洁、清晰、可直接用于 Commit。
4. 输出格式严格为：

`[type]: 完成指定功能/修复问题/优化结构`

5. 根据实际修改内容，将模板中的描述替换为准确中文短语（10-20 字优先）。

## 类型自动判断规则

- `feat`：新增功能、新增模块、新增接口、新增流程。
- `fix`：修复错误、修复异常、修复逻辑缺陷、修复兼容性问题。
- `refactor`：重构代码结构、提取公共逻辑、优化实现但不改变功能行为。
- `style`：仅样式/格式/命名/注释调整，不涉及功能逻辑变化。
- `chore`：工程维护类改动，如脚本、配置、依赖、构建流程、目录整理。

## 输出示例

- `feat: 新增故障诊断训练流程入口`
- `fix: 修复CWT数据加载维度错误`
- `refactor: 重构CNN模型训练与评估流程`
- `style: 统一数据处理模块命名与注释`
- `chore: 更新训练脚本配置与路径管理`

## 约束

- 不输出多条提交信息。
- 不输出与提交无关的解释性文本作为最终行。
- 若变更跨多类型，优先选择“对用户价值最大”的主类型。
